# Domain Research: news（新闻事件类）

## 1. 風格核心定義

**本質目標**: 客観的な財経事件・マーケットイベントを、事実ベースで迅速かつ正確に伝達すること。読者が「何が起きたか」を最短時間で把握し、次のアクション判断に移れることがゴール。

**他風格との核心区別**:
- vs `report`（情報分析）: report は「なぜ起きたか・今後どうなるか」を分析・推測する。news は「何が・いつ・どこで起きたか」の事実伝達に特化。hedging 表現は見通し文のみ、本文は断定的事実記述。
- vs `marketing`（営業推広）: marketing は読者の行動変容を目的とし CTA 必須。news は情報提供が目的で CTA は原則不要（関連記事リンク程度）。
- vs `topic`（コミュニティ投稿）: topic はエンゲージメント（コメント・投票）を狙う。news は客観報道に徹し読者参加要素を持たない。

## 2. 目標受众与触达时机

| 受众セグメント | 触達タイミング | 期待される行動 |
|---|---|---|
| アクティブ個人投資家 | 市場開場前/引け後、速報発生時 | 情報を基にポジション判断 |
| 情報収集段階の投資初心者 | 日次/週次定期配信 | 市場理解の深化、プラットフォームへの信頼構築 |
| 高頻度トレーダー | イベント発生直後（分単位） | 即時の売買判断材料として利用 |
| 潜在ユーザー（未口座開設） | SNS/ニュースフィード経由 | プラットフォームの情報力を認知→口座開設動機 |

**触達チャネル**: Push通知、アプリ内ニュースフィード、LINE配信、メールダイジェスト、SNS投稿、Webマガジン

## 3. 語体/レジスタ特征

| 要素 | 仕様 |
|------|------|
| 動詞詞尾 | だ/である体（速報・本文）; です/ます体（ダイジェスト・初心者向け解説） |
| 敬語等級 | 原則なし（客観報道体）; ダイジェストメール等ユーザー向けではです/ます |
| 句長 | 速報: 15–25字/文; 本文記事: 25–40字/文; ダイジェスト: 20–35字/文 |
| 語気 | 断定的事実記述（「〜した」「〜となった」）、見通しのみ hedging |
| 禁止表現 | 主観（思う/感じる）、感嘆符（！）、CTA動詞、煽り表現 |
| 数値処理 | 原数値厳守; 変動率は前日比/前期比を明記; 出典不要（公知データ） |
| 時制 | 既発生事象=過去形; 予定=「〜する見通し」「〜の予定」 |

## 4. 日本金融行業の典型載体

| 媒介/フォーマット | 参考ソース | 特徴 |
|---|---|---|
| アプリ内速報フィード | moomoo証券「ニュース」、楽天証券「マーケットニュース」 | 1–3文の超短文、数値先出し、タイムスタンプ付き |
| 市況まとめ記事 | 野村証券「マーケット・エクスプレス」、SBI証券「マーケットレポート」 | 300–600字、セクション分け（株式/為替/債券） |
| 決算速報 | 各証券会社 決算フラッシュ | 売上/利益/予想対比を表形式+1–2文コメント |
| 経済指標カレンダー解説 | 楽天証券「経済指標」、みんかぶ | 指標名+結果+予想+前回値+1文解説 |
| Push/LINE速報 | moomoo証券Push通知 | タイトル≤20字+本文≤60字（news版は事実のみ、CTAなし） |
| 週次/月次ダイジェストメール | 野村証券 週間マーケットレビュー | 5–7トピックを各2–3文で要約、です/ます体 |

## 5. 推薦 Tier 2 子場景

| Scene ID | 名称 | 説明 | 典型字数 |
|---|---|---|---|
| `flash` | 速報・フラッシュ | 事件発生直後の1–3文速報。数値+事実のみ、解説なし。 | ≤100字 |
| `wrap` | 市況まとめ | 1日/1週間のマーケット動向を構造的に要約。セクション分け。 | 300–600字 |
| `earnings` | 決算サマリー | 個別企業の決算結果を定型フォーマットで伝達。 | 100–200字 |
| `indicator` | 経済指標解説 | 経済指標の結果と市場インパクトを短文で解説。 | 80–150字 |
| `digest` | ニュースダイジェスト | 複数トピックを集約した定期配信用要約。です/ます体。 | 400–800字 |
| `alert` | 速報Push/LINE | Push通知・LINE向け超短文ニュース。タイトル+本文2行構造。 | タイトル≤20字; 本文≤60字 |

## 6. 関鍵約束草案

| # | 約束 | 根拠 |
|---|------|------|
| NC-1 | **事実先出し（Fact-First）**: 第1文は必ず核心事実（数値/事象）を含む。背景・解説は後続文。 | 速報性：読者は最初の1文で判断する |
| NC-2 | **数値完全保持**: 入力に含まれる数値（株価、変動率、日付、金額）は一切変更せず出力に保持する。単位変換（円→ドル等）禁止。 | RL-1 拡張：news における数値は term と同等の lock 対象 |
| NC-3 | **主観ゼロ・煽りゼロ**: 「思う」「感じる」「チャンス」「注目」「期待」等の主観/煽り表現禁止。見通しは「〜の見込み」「〜と見られる」のみ。 | RL-3 準拠 + news 固有の客観性要件 |
| NC-4 | **時間属性必須**: 速報/フラッシュでは日付またはタイムスタンプを含む。まとめ記事では対象期間を明記。 | 読者のコンテクスト把握に必須 |
| NC-5 | **CTA 禁止（原則）**: news 本文には行動喚起（「今すぐ確認」「口座開設」等）を含めない。関連記事へのリンク示唆（「詳細はこちら」）のみ許容。 | report/marketing との境界維持 |
| NC-6 | **出典処理**: 公知データ（指数水準、中央銀行決定等）は出典不要。アナリスト予測・第三者調査は「〜によると」を付す。 | RL-9 の news 版：公知 vs 非公知の区分 |
| NC-7 | **レジスタ純度**: 速報/まとめ=だ/である体で統一; ダイジェスト=です/ます体で統一。混合禁止（RL-11）。 | register mixing 防止 |

## 7. 隣接風格との境界

### news vs report（情報分析）

| 軸 | news | report |
|---|---|---|
| 目的 | 事実伝達 | 分析・予測 |
| hedging 密度 | 見通し文のみ（1–2文/記事） | 全文にわたり推測表現が散布 |
| 著者の立場 | 記者（事実報告） | アナリスト（解釈提示） |
| 構造 | 逆三角形（重要度順） | 背景→分析→見通し |
| 判定基準 | 「なぜ・今後どうなるか」の分析パートが全体の30%以下→news | 分析パートが30%超→report |

### news vs marketing（営業推広）

| 軸 | news | marketing |
|---|---|---|
| CTA | なし（リンク示唆のみ） | 必須 |
| 語気 | 客観・断定的 | 読者利益訴求・能動態 |
| 数値の扱い | 事実としてそのまま提示 | ベネフィットとしてフレーミング |
| 判定基準 | 読者に特定行動を求めているか？ No→news / Yes→marketing |

### news vs topic（コミュニティ投稿）

| 軸 | news | topic |
|---|---|---|
| 読者参加 | なし | 投票/コメント/投稿例あり |
| レジスタ | だ/である（客観報道体） | です/ます（エンゲージメント体） |
| 目的 | 情報伝達 | エンゲージメント駆動 |
| 判定基準 | 読者とのインタラクション要素があるか？ No→news / Yes→topic |

## Open Questions

- flash と alert の統合可否：両者とも超短文だが、flash=フィード表示（全文可視）、alert=Push（タイトル/本文分離）で構造が異なるため分離推奨。
- digest のレジスタ：ユーザー向けメールではです/ます体が自然だが、report T1 のだ/である制約と衝突する。news を独立 T1 にすることでこの問題を解消。
- earnings の表形式：決算サマリーは表（売上/利益/予想対比）+短文コメントの混合フォーマット。Markdown 表記対応が必要。
