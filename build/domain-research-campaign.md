# Domain Research: campaign（营销活动类）

## 1. 風格核心定義

**本質目標**: 特定の期間・条件を持つ短期激励活動（キャンペーン・プロモーション）を通じて、ターゲットセグメントの即時行動（口座開設、入金、取引開始、再活性化）を最大化すること。「限られた時間内に、特定の条件を満たして、特典を獲得する」という行動ループを設計・伝達するコンテンツ全般。

**他風格との核心区別**:
- vs `marketing`（営業推広）: marketing は恒常的なブランド訴求・プロダクト価値のコンバージョン最適化（LP、SNS常時運用等）。campaign は**期間限定・条件付き・特典駆動**の短期バースト施策に特化。marketing が「この商品はなぜ良いか」を伝えるのに対し、campaign は「今やると何がもらえるか」を伝える。
- vs `trade-ideas`（赚效内容类）: trade-ideas は市場分析・投資アイデアを通じた取引動機付け（Logos/情報価値主導）。campaign は特典・報酬を通じた行動動機付け（Pathos/インセンティブ主導）。情報で動かすか、報酬で動かすかの違い。
- vs `product`（産品説明）: product は機能理解・定着が目的で時間依存性が低い。campaign は時限性が本質（期間外は意味を持たない）。
- vs `report`（情報分析）: report は客観的市場分析。campaign は主観的・訴求的トーンが許容され、読者の行動変容が直接目標。
- vs `legal`（法律合規）: campaign 文案には法的制約（景品表示法、金商法広告規制）が内包されるが、legal 風格のリスク警告構文ではなく、条件表示の正確性・透明性で対応する。

## 2. 目標受众与触达时机

| 受众セグメント | 触達タイミング | 期待される行動 | 心理トリガー |
|---|---|---|---|
| 未口座開設者（コールドリード） | 市場活況期、決算シーズン、新NISA開始時 | 口座開設完了 | low_barrier, incentive, social_proof |
| 開設済み未入金者 | 開設後7–30日、キャンペーン期間 | 初回入金 | loss_aversion (期限), incentive |
| 開設済み未取引者（never_traded） | 市場変動時、ボーナス時期 | 初回取引執行 | low_barrier, curiosity_gap, fomo |
| 休眠ユーザー（churn/inactive） | 離脱後30–90日、市場転換点 | 復帰ログイン→再取引 | loss_aversion, exclusivity, incentive |
| 単一商品ユーザー（cross-sell） | 新商品リリース時、関連市場イベント | 新商品カテゴリ取引開始 | opportunity, fomo, incentive |
| 高資産ユーザー（high_value） | VIPプログラム更新時、大型IPO時 | 追加入金/高額取引 | exclusivity, performance, status |

**触達チャネル**: Push通知、EDM（メール）、アプリ内バナー、SNS広告、LP、アプリ内ポップアップ、LINE公式アカウント、SMS

**触達時機の特殊性**: campaign は「開始告知→リマインド→最終日アラート→終了報告」のライフサイクルを持ち、同一キャンペーンで時期に応じたメッセージ変容が必要。

## 3. 語体/レジスタ特征

| 要素 | 仕様 |
|------|------|
| 動詞詞尾 | です/ます体（基本）; Push タイトル・バナーメインは体言止め許容 |
| 敬語等級 | 丁寧語（です/ます）; 過度な謙譲語不要; SNS/Push はカジュアル寄り許容 |
| 句長 | Push: ≤20/≤60字; バナー: ≤15/≤25字; SNS: ≤140字; EDM件名: ≤40字; LP/event: 15–25字/文 |
| 語気 | **緊迫感+具体性**: 期限・数量・条件を明示しつつ行動を促す; 「！」使用許容（1–2回/ブロック） |
| 情報優先順位 | 特典内容 → 条件 → 期限 → CTA（キャンペーン本体）; 期限 → 特典 → CTA（リマインド） |
| 数値表現 | 金額は「相当」付加（「最大5万円相当」）; 確率表現は「必ずもらえる」形式; 具体数字先出し |
| CTA形式 | Hot stage基本:「今すぐ申し込む」「参加する」; セグメント別CTA stage alignment適用 |
| 禁止表現 | 「100%当選」（→「必ずもらえる」）、曖昧な特典表現（「素敵な特典」→具体名称）、根拠なき最上級 |
| ブランド表記 | 本文: 正式名称（「moomoo証券」）; 件名/タイトル: 文脈上明確な場合省略可 |
| キャンペーン名 | 【】括弧で囲む（件名・Push タイトル）; 本文内は「」括弧 |

## 4. 日本金融行業の典型載体

| 媒介/フォーマット | 参考ソース | 特徴 |
|---|---|---|
| キャンペーンLP | 楽天証券「口座開設キャンペーン」、SBI証券「新規口座開設特典」 | 特典ヒーロー→条件一覧→ステップ→CTA; 期限カウントダウン付き |
| Push通知（キャンペーン告知） | moomoo証券アプリ通知 | タイトル≤20字（数字/特典名先出し）+本文≤60字; 開始/リマインド/最終日の3パターン |
| EDM（キャンペーンメール） | 楽天証券メルマガ、SBI証券メール | 【キャンペーン名】件名→プリヘッダー→条件/特典→CTA; 複数キャンペーン並列の場合あり |
| SNSキャンペーン投稿 | 各証券会社Twitter/X | ≤140字; 🎉📈絵文字; 期限+特典+CTA; RT/いいねキャンペーン連動 |
| アプリ内バナー | moomoo証券・楽天証券アプリ | メイン≤15字+サブ≤25字+CTA≤8字; 視覚インパクト最優先 |
| アプリ内ポップアップ/モーダル | 各証券アプリ | 特典名+条件+CTA; 閉じるボタン必須; 表示頻度制御あり |
| LINE公式メッセージ | 野村証券LINE、SBI証券LINE | カード形式 or テキスト; 会話的トーン;「チェックしてみよう」CTA |
| キャンペーン結果通知 | 各社アプリ/メール | 達成報告:「おめでとうございます！」; 未達成:「惜しい！次回も」; 特典付与確認 |

## 5. 推薦 Tier 2 子場景

| Scene ID | 名称 | 説明 | 典型字数 | 情報構造 |
|---|---|---|---|---|
| `launch` | キャンペーン開始告知 | 新キャンペーンの初回告知。特典内容と条件を明確に伝え、参加を促す。 | LP: 300–800字; Push/SNS: 制限内 | 特典名称→条件→期限→CTA |
| `remind` | リマインド・中間アラート | 期間中の再訴求。進捗状況や残り時間を強調し、未参加者/未完了者を喚起。 | Push: ≤80字; EDM: 200–400字 | 残り期間→現在状況→行動促進→CTA |
| `lastday` | 最終日・ラストチャンス | 期限直前の緊急訴求。最大の緊迫感で即時行動を駆動。 | Push: ≤80字; SNS: ≤140字 | 「本日最終」→特典→CTA（最ホット） |
| `result` | 結果通知・達成報告 | キャンペーン結果の個別通知。達成/未達成/特典付与を明確に伝達。 | Push: ≤80字; アプリ内: 100–200字 | 結果→特典内容→次のステップ |
| `seasonal` | 季節/イベント連動型 | 年末年始、新NISA開始、決算シーズン等、市場イベントに紐づくキャンペーン。 | LP: 400–1000字; 全チャネル展開 | 市場コンテクスト→特典→条件→期限→CTA |
| `referral` | 紹介・友達招待型 | ユーザー間の紹介を通じた新規獲得施策。双方向特典の明示が核。 | SNS/EDM: 200–400字 | 紹介特典（双方）→条件→紹介方法→CTA |

## 6. 関鍵約束草案

| # | 約束 | 根拠 |
|---|------|------|
| CA-1 | **期限明示必須**: すべての campaign コンテンツに具体的な終了日時または期間を明記。「期間限定」だけでは不十分、「〜月〜日まで」を必須とする。 | 行動経済学の deadline effect; 景品表示法の期間表示義務 |
| CA-2 | **条件透明性**: 特典獲得の条件（対象者・必要行動・最低金額等）を省略なく記載。細字/注釈ではなく本文レベルで伝達。 | 金融庁ガイドライン: 重要条件の視認性確保 |
| CA-3 | **特典具体化**: 特典名称は具体的に記載（「株式キャッシュバック券」「最大5万円相当ポイント」）。「お得な特典」「素敵なプレゼント」等の抽象表現禁止。 | 具体性が行動率を高める; 景品表示法の有利誤認防止 |
| CA-4 | **金額表現ルール**: 金額には「相当」を付加（「5万円相当」）; 確率は「必ずもらえる」形式; 「100%当選」禁止; 「賞品を山分け」（not「賞金プール」）。 | compliance 準拠; marketing T1 の Prize amount hedging ルール継承 |
| CA-5 | **CTA セグメント整合**: CTA の温度感はターゲットセグメントの funnel stage に整合。Cold:「まずは詳しくみる」/ Warm:「参加する」/ Hot:「今すぐ申し込む」。 | 行動心理学: コミットメント段階に応じた CTA が最も効果的 |
| CA-6 | **ライフサイクル一貫性**: 同一キャンペーンの launch → remind → lastday → result で、キャンペーン名・条件・特典表記を完全統一。表記揺れ禁止。 | ユーザー認知の連続性確保; 混乱によるドロップオフ防止 |
| CA-7 | **マルチステート整合**: 参加前/達成/未達成/期限切れ等の状態別メッセージで、レジスタ・トーンを統一。達成通知がカジュアルで未達成通知が硬い、等のトーンシフト禁止。 | ブランド体験の一貫性; RL-11 精神の拡張 |
| CA-8 | **煽り表現の適正化**: 緊迫感表現は許容するが、事実に基づく形式のみ（「残りX日」「先着X名」）。根拠のない煽り（「チャンス密集中」「仕込みのタイミング」）は禁止。 | marketing T1 の Market context in titles/closers ルール; 景品表示法 |

## 7. 隣接風格との境界

### campaign vs marketing（営業推広）

| 軸 | campaign | marketing |
|---|---|---|
| 時間軸 | 期間限定（開始日〜終了日あり） | 恒常的（常時掲載可能） |
| 主動機 | インセンティブ（特典/報酬） | プロダクト価値/ブランド |
| 情報核心 | 特典+条件+期限 | ベネフィット+差別化+CTA |
| 緊迫感 | 必須（deadline effect 活用） | 任意（urgency は補助的） |
| CTA 温度 | 基本 Hot（「申し込む」「参加する」） | セグメント/場面依存 |
| ライフサイクル | 開始→リマインド→最終日→結果 | 単発または常時 |
| 判定基準 | 「期間限定の特典/条件がある」→campaign / 「恒常的な商品価値訴求」→marketing |

### campaign vs trade-ideas（赚效内容类）

| 軸 | campaign | trade-ideas |
|---|---|---|
| 動機付け手段 | 外的報酬（ポイント/現金/特典） | 内的報酬（投資収益/知的好奇心） |
| コンテンツ性質 | 条件/特典の伝達（情報量少、CTA重） | 市場分析/投資アイデアの提供（情報量多、教育的） |
| 主導証明 | Pathos（緊迫感、得する感情） | Logos（データ、分析根拠） |
| 行動目標 | キャンペーン参加（入金/取引/口座開設） | 取引実行（特定銘柄/商品の売買） |
| 時間依存性 | キャンペーン期間に強く依存 | 市場タイミングに依存（ホットトピック連動） |
| 判定基準 | 「特典をもらうために行動させる」→campaign / 「投資で稼ぐために行動させる」→trade-ideas |

### campaign vs product（産品説明）

| 軸 | campaign | product |
|---|---|---|
| 目的 | 即時行動（参加・申込） | 理解・活用促進 |
| 時間依存性 | 高（期間外は無効） | 低（恒常的な情報） |
| トーン | 訴求的・緊迫的 | 説明的・教育的 |
| CTA | 必須（Hot stage基本） | 任意（低コミットメント） |
| 判定基準 | 「期限付きの行動喚起」→campaign / 「機能・使い方の説明」→product |

## Open Questions

- `seasonal` と `launch` の境界: 季節イベント連動型は市場コンテクストの導入部が長くなり trade-ideas に近づく場合がある。→ 提案: 主目的が「特典獲得の行動喚起」なら campaign/seasonal、「市場情報提供→取引動機」なら trade-ideas。
- キャンペーン横断型LP（複数キャンペーンを1ページにまとめた「キャンペーン一覧」）の扱い: → 提案: 個別キャンペーンの campaign/launch を並列配置する構造として、campaign 内で対応。
- `result`（結果通知）は support/guide に近いニュアンスもある（手続き説明的）: → 提案: 主目的が「達成祝福+次の行動喚起」なら campaign/result。「特典の受取手順説明」が主なら product/manual。
