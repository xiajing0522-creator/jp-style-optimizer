# Domain Research: trade-ideas（赚效内容类）

## 1. 風格核心定義

**本質目標**: 市場のホットトピック・投資テーマ・銘柄分析を活用し、異なる投資経験レベルのユーザーに対して「取引したい」という内的動機を喚起し、実際の売買行動（取引転化）に結びつけること。外的インセンティブ（特典/ポイント）ではなく、「この取引で利益を得られる可能性」という情報価値で行動を駆動する。

**他風格との核心区別**:
- vs `report`（情報分析）: report は純粋な客観分析（三人称・推測表現必須・CTA禁止）。trade-ideas は分析を**行動への橋渡し**として使う。report が「何が起きているか」を伝えるのに対し、trade-ideas は「だからどう取引すべきか/できるか」を示唆する。trade-ideas は客観性を維持しつつも、読者の取引行動を意識した情報設計を行う。
- vs `campaign`（营销活动类）: campaign は外的報酬（特典/ポイント）で動機付け。trade-ideas は内的報酬（投資リターンの期待・知的満足）で動機付け。campaign が「もらえるから動く」なら、trade-ideas は「稼げるかもしれないから動く」。
- vs `marketing`（営業推広）: marketing は商品/サービスの恒常的価値訴求。trade-ideas は**今この瞬間の市場機会**に紐づく時限的な取引提案。marketing が「なぜこの証券会社が良いか」を伝えるのに対し、trade-ideas は「なぜ今この取引が面白いか」を伝える。
- vs `product`（産品説明）: product は機能説明・操作方法。trade-ideas は投資判断の材料提供。product の「こう使える」に対し trade-ideas は「こう取引できる」。
- vs `legal`（法律合規）: trade-ideas は投資助言に該当しないよう、必ず「情報提供目的」の免責を付す。判断は読者自身に委ねる表現が必須。

## 2. 目標受众与触达时机

| 受众セグメント | 触達タイミング | 期待される行動 | 心理トリガー |
|---|---|---|---|
| 投資初心者（口座あり・取引少） | 分かりやすいテーマ相場時（新NISA、高配当等） | テーマ理解→初回/少額取引 | curiosity_gap, social_proof, low_barrier |
| 日本株中心トレーダー | 米国株/新興国の話題沸騰時 | 新市場への取引拡大 | opportunity, fomo, diversification |
| 米国株中心トレーダー | 日本株の独自材料出現時（円安、PBR改革等） | 日本株取引開始 | opportunity, fomo |
| テクニカル派トレーダー | ボラティリティ急騰時、重要テクニカル水準接近時 | 積極的な売買（短期） | pattern_recognition, urgency |
| 長期投資家 | 大幅下落後のバリュエーション改善時 | 押し目買い/積増し | value_opportunity, contrarian |
| 休眠トレーダー | 大きな市場イベント（暴落/暴騰/IPO） | 復帰ログイン→取引再開 | fomo, loss_aversion, curiosity_gap |

**触達チャネル**: アプリ内フィード（投資情報タブ）、Push通知、コミュニティ投稿（Topic記事）、EDM（市場速報メール）、SNS、動画コンテンツ（YouTube/アプリ内）

**触達時機の特殊性**: trade-ideas は**市場イベント駆動**。決算発表、経済指標、地政学イベント、テーマ株の急騰/急落など、特定の市場タイミングでのみ有効性が最大化する。速報性（数時間以内の配信）が成否を分ける。

## 3. 語体/レジスタ特征

| 要素 | 仕様 |
|------|------|
| 動詞詞尾 | **二重レジスタ制**: 分析パート=だ/である体（report T1 準拠）; 読者アクションパート=です/ます体（engagement促進） |
| 敬語等級 | 分析パート: 敬語なし（客観体）; アクションパート: 丁寧語（です/ます） |
| 句長 | 速報型: 15–25字/文; 分析型: 25–40字/文; コミュニティ型: 25–50字/文 |
| 語気 | **知的興奮+客観性のバランス**: 「注目されている」「関心が集まっている」は許容; 「絶対上がる」「必ず儲かる」は禁止 |
| 数値表現 | 株価・騰落率・PER等は正確に保持; 比較データは前日比/前年比で文脈化; 出典明記（RL-9 準拠、ただし広く報道されたデータは免除） |
| 推測表現 | 必須: 「〜と見られる」「〜の可能性がある」「〜が注目される」; 断定禁止; confidence gradient 適用 |
| CTA 形式 | 間接的: 「〜をチェックしてみませんか」「〜の値動きに注目です」; 直接的:「取引画面を見る」「銘柄詳細へ」; 投資助言的CTA禁止（「今すぐ買おう」） |
| 禁止表現 | 「必ず上がる」「儲かる」「おすすめ銘柄」（投資助言該当リスク）; 主観的断定; 煽り的表現（「爆益」「神銘柄」）※コミュニティ投稿例を除く |
| 免責 | 長文コンテンツには末尾に免責文言を付記: 「本情報は投資助言を目的としたものではありません」 |

## 4. 日本金融行業の典型載体

| 媒介/フォーマット | 参考ソース | 特徴 |
|---|---|---|
| 市況速報（ホットトピック） | 楽天証券「マーケット情報」、moomoo証券「ホットトピック」 | ≤140字; 事実+影響+見通し; だ/である体; 速報性最重視 |
| 投資テーマ記事 | SBI証券「テーマ株」、野村証券「市場見通し」 | 500–1500字; テーマ背景→関連銘柄→投資視点; 中立的分析体 |
| コミュニティ話題記事（Topic） | moomoo証券コミュニティ | 400–800字; です/ます体; 読者参加型（投稿例+投票）; 既存 report/topic scene |
| 銘柄分析カード | moomoo証券・楽天証券アプリ | 100–200字; 銘柄名+注目理由+基本指標; カード形式で一覧性重視 |
| Push通知（市場速報） | 各証券アプリ | タイトル≤20字+本文≤60字; 「日経平均○○円突破」「決算サプライズ」等; 速報→アプリ誘導 |
| 朝の市場ブリーフ/デイリーレポート | 野村証券「朝の一言」、大和証券「デイリーレポート」 | 300–800字; 前日振り返り→本日注目点→注目銘柄/指標; だ/である体 |
| 決算ハイライト | 各証券会社 IR・決算速報 | 200–500字; EPS/売上等の実績vs予想→評価→株価反応; データ密度高 |
| 動画マーケット解説 | moomoo証券YouTube、楽天証券「トウシル」 | 口語体≤30字/文; チャート参照しながらの解説; script T2 準拠 |
| EDM（市場機会メール） | 各証券会社メルマガ | 件名: 注目テーマ/銘柄→本文: 分析サマリー→CTA:「詳しく見る」; 週次/不定期 |

## 5. 推薦 Tier 2 子場景

| Scene ID | 名称 | 説明 | 典型字数 | 情報構造 |
|---|---|---|---|---|
| `flash` | 市場速報・フラッシュ | 市場の重要イベント（急騰/急落/政策変更/決算サプライズ）の即時配信。速報性と簡潔性が命。 | ≤140字（Push/SNS用も同時生成） | 事実（数値付き）→影響→見通し |
| `theme` | テーマ分析・投資アイデア | 特定の投資テーマ（AI、高配当、インバウンド等）を深掘りし、関連銘柄/商品を紹介。中長期視点。 | 500–1500字 | テーマ背景→市場データ→関連銘柄/商品→投資視点→免責 |
| `earnings` | 決算ハイライト | 個別銘柄の決算結果を速報的に分析。実績vs予想、株価反応、今後の見通し。 | 200–500字 | 決算概要（実績vs予想）→ポイント分析→株価反応→見通し |
| `morning` | 朝のブリーフ・デイリー | 前日の市場振り返りと本日の注目ポイント。毎朝の定型配信。 | 300–800字 | 前日振り返り（主要指標）→本日注目材料→注目銘柄/セクター |
| `picks` | 注目銘柄カード | 短尺の銘柄紹介。カード形式で複数銘柄を一覧的に紹介。 | 80–200字/銘柄 | 銘柄名→注目理由（1–2文）→基本指標→関連テーマタグ |
| `community` | コミュニティ投資話題 | 既存 report/topic scene の拡張。市場テーマを読者参加型コンテンツに変換。 | 400–800字+参加モジュール | 既存 topic scene（Variant A）構造に準拠 |

## 6. 関鍵約束草案

| # | 約束 | 根拠 |
|---|------|------|
| TI-1 | **投資助言非該当の徹底**: 「おすすめ」「買い/売り推奨」「必ず上がる」等の助言的表現を一切禁止。あくまで情報提供・分析紹介の体裁を維持。 | 金融商品取引法: 投資助言業登録なしでの助言行為は違法 |
| TI-2 | **推測表現の義務化**: すべての前向き（forward-looking）記述に推測表現を付す。confidence gradient 適用（見込み > と見られる > 可能性がある）。 | report T1 共通制約の継承; 断定的判断の提供は金商法違反リスク |
| TI-3 | **データ出典の明示**: 具体的数値（株価、騰落率、EPS、PER等）には出典を付記。ただし広く報道済みのデータ（指数水準、前日終値等）は免除。 | RL-9 準拠; 情報の信頼性担保 |
| TI-4 | **市場タイミング整合**: コンテンツの鮮度を明示（「本日時点」「X月X日終値ベース」）。過去データに基づく分析は日付を明記。 | 投資判断に用いられる情報は時間依存性が極めて高い |
| TI-5 | **行動橋渡し設計**: 分析パートの後に必ず「次のステップ」を示唆する構造を持たせる。直接的CTA（「取引画面へ」）または間接的示唆（「〜に注目です」「〜をチェック」）。 | trade-ideas の本質目標=取引転化; 分析だけで終わると report と差別化できない |
| TI-6 | **リスク両面提示**: ポジティブな投資テーマを紹介する際、必ずリスク要因/下落シナリオにも言及。片面提示禁止。 | 金融庁ガイドライン: バランスのとれた情報提供義務; 投資助言非該当の補強 |
| TI-7 | **免責文言の必須付記**: 500字を超えるコンテンツ（theme/morning/community等）には末尾に標準免責文言を付記。 | 金商法広告規制; 投資助言非該当の形式的担保 |
| TI-8 | **主観排除・知的興奮の両立**: 「注目されている」「関心が集まっている」「話題を呼んでいる」等の客観的注目度表現は許容。「面白い」「すごい」「ヤバい」等の主観的評価は禁止（community の投稿例パートを除く）。 | RL-3 精神の継承; 客観性を維持しつつ engagement を確保する手段 |

## 7. 隣接風格との境界

### trade-ideas vs report（情報分析）

| 軸 | trade-ideas | report |
|---|---|---|
| 最終目的 | 取引行動の喚起 | 情報提供・分析共有 |
| CTA/行動示唆 | 必須（間接的〜直接的） | 禁止（純粋情報提供） |
| レジスタ | 二重レジスタ（分析:だ/である + 行動:です/ます） | 単一レジスタ（だ/である統一、topic のみ例外） |
| リスク提示 | 必須（両面提示で助言非該当を担保） | 必須（客観分析の一部として） |
| 免責文言 | 長文に必須付記 | 原則不要（そもそも客観体で助言性なし） |
| 読者参加 | 許容（community sub-scene で投票/コメント促進） | topic scene でのみ許容 |
| 判定基準 | 「分析を踏まえた取引行動を期待」→trade-ideas / 「分析それ自体が価値」→report |

### trade-ideas vs campaign（营销活动类）

| 軸 | trade-ideas | campaign |
|---|---|---|
| 動機付け | 内的報酬（投資リターン期待） | 外的報酬（特典/ポイント） |
| 時間依存性 | 市場タイミング依存（数時間〜数日） | キャンペーン期間依存（数日〜数週間） |
| コンテンツ密度 | 高（データ/分析/銘柄情報） | 低（条件/特典/CTA） |
| 主導証明 | Logos（データ・分析） | Pathos（緊迫感・得する感情） |
| 免責 | 投資助言非該当の免責 | 景品表示法/条件表示の正確性 |
| 判定基準 | 「市場情報で取引動機を喚起」→trade-ideas / 「特典で行動動機を喚起」→campaign |

### trade-ideas vs marketing（営業推広）

| 軸 | trade-ideas | marketing |
|---|---|---|
| フォーカス | 市場機会（「今何が面白いか」） | 商品価値（「なぜこの証券会社が良いか」） |
| 時間軸 | 時限的（市場イベント連動） | 恒常的（常時掲載可能） |
| CTA の性質 | 取引行動促進（「銘柄を見る」「取引画面へ」） | コンバージョン促進（「口座開設」「申し込む」） |
| コンテンツ価値 | 情報価値（分析・データ） | 訴求価値（ベネフィット・差別化） |
| 判定基準 | 「市場情報を提供して取引を促す」→trade-ideas / 「サービス価値を伝えてコンバージョンを促す」→marketing |

### trade-ideas vs product（産品説明）

| 軸 | trade-ideas | product |
|---|---|---|
| 主題 | 市場/銘柄/投資テーマ | プロダクト機能/使い方 |
| 時間依存性 | 高（市場環境で有効性変動） | 低（機能は恒常的） |
| 行動目標 | 売買実行 | 機能活用・理解 |
| 判定基準 | 「何を取引するか」の情報→trade-ideas / 「どう取引するか」の情報→product |

## Open Questions

- `flash`（市場速報）と既存 report/hottopic の関係: hottopic は 140 字制限の純粋速報。flash は trade-ideas の文脈で「速報→行動示唆」を含むため、構造的に hottopic + CTA 示唆となる。→ 提案: flash は hottopic を拡張した形式で、末尾に間接的行動示唆（「〜に注目」「〜をチェック」）を付加可能とする。
- `community` と既存 report/topic の関係: topic scene は既に report T1 傘下で定義済み。trade-ideas/community は topic の構造を流用しつつ、行動橋渡し（TI-5）の意識をより強化する位置づけ。→ 提案: 既存 topic scene をそのまま活用し、trade-ideas T1 の制約（TI-1〜TI-8）を追加適用する形で統合。
- レジスタの二重性: trade-ideas は「分析パート（だ/である）」と「読者向けパート（です/ます）」が混在する。RL-11（レジスタ混在禁止）との整合が必要。→ 提案: RL-11 の例外規定として、trade-ideas T1 では「セクション単位でのレジスタ分離」を許容。同一段落内の混在は禁止。
- 投資助言との境界: trade-ideas の「行動橋渡し」が投資助言に踏み込まないためのガードレール定義が重要。→ 提案: 「特定銘柄の売買を直接推奨する表現」を RL レベルで禁止し、あくまで「情報提供+読者自身の判断」のフレームを維持。
