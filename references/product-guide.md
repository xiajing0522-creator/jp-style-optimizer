# Product Guide（产品运营类）

## T1 Shared Constraints

**Register**: です/ます体統一。
**Dominant proof**: Logos（機能・仕様・実績数値）。Pathos禁止。
**CTA**: 任意（低コミットメント形のみ）。「今すぐ申し込む」等の高圧CTAは禁止（campaign領域）。

| Constraint | Rule | Verification |
|-----------|------|-------------|
| PC-1 F→B必須ペア | 機能記述には必ず対応ユーザーメリットを付す | PC-1 check |
| PC-2 専門用語初出説明 | 金融/技術用語の初出時に括弧内で平易説明。2回目以降不要 | PC-2 check |
| PC-3 誇張禁止 | 「業界最安」「No.1」等に具体的根拠（出典/数値/条件）必須。根拠なき断定禁止 | PC-3 scan |
| PC-4 CTA任意 | CTA付す場合は「詳しくはこちら」「試してみる」等の低圧形。「今すぐ申し込む」禁止 | PC-4 check |
| PC-5 1アクション/1ステップ | manual/onboardの手順記述: 1ステップ=1アクション。UI要素名は「」括弧 | PC-5 check (manual/onboard) |
| PC-6 スキャナビリティ | 3項目以上の並列情報は箇条書き/表形式。4文以上の連続散文ブロック禁止（manual/detail） | PC-6 check |
| PC-7 能動態優先 | 受動態を能動態に変換。ただしmanualで主語隠し目的の受動態は許容 | PC-7 scan |
| PC-8 数値具体化 | 「高速」「簡単」→「約3秒」「3ステップ」等に具体化。入力数値は必ず保持 | PC-8 check |

## T2 Scene Specifications

### short（短産品文案・USPカード）
**字数**: 30–80字
**情報構造**: 卖点 → 差異化 → CTA（任意）
**Emphasis**: 単一セリングポイント。1文で完結。
**Checkpoint**: 1機能+1ベネフィット ✓ / 80字以内 ✓

### detail（詳細説明ページ）
**字数**: 200–500字/セクション
**情報構造**: 機能名 → ベネフィット → 説明 → CTA（任意）
**必須**: F→B必須ペア（PC-1）; 3項目以上は箇条書き（PC-6）
**Checkpoint**: 全機能にF→B ✓ / 散文ブロック≤3文 ✓

### manual（使用説明・手順書）
**字数**: 150–400字
**情報構造**: 目的 → 前提条件 → 手順（番号付き）→ 注意事項
**必須**: 1ステップ1アクション（PC-5）; UI要素は「」
**Checkpoint**: 番号付きステップ ✓ / 複合アクション不在 ✓

### script（動画・音声スクリプト）
**字数**: 200–600字
**情報構造**: 導入（フック）→ 説明 → まとめ/CTA
**Register**: 口語体。文長≤30字/文。書き言葉禁止
**Checkpoint**: 全文≤30字 ✓ / 書き言葉不在 ✓

### release（リリースノート・更新情報）
**字数**: 100–300字
**情報構造**: バージョン/日付 → 変更点リスト → 影響 → 手順（あれば）
**Emphasis**: 変更点は箇条書き。「改善」「追加」「修正」で明示
**Checkpoint**: バージョン/日付明記 ✓ / 箇条書き構造 ✓

### onboard（オンボーディング）
**字数**: 200–500字
**情報構造**: ゴール提示 → ステップ（番号付き）→ 達成確認
**必須**: プロアクティブ（困り前）。手順はPC-5。達成確認文を末尾に
**Checkpoint**: ゴール明示 ✓ / 達成確認あり ✓

### compare（比較表・USP対比）
**字数**: 100–200字（表除く）
**情報構造**: 比較軸 → 表 → 補足1–2文
**必須**: PC-3（比較優位には根拠）; 表形式
**Checkpoint**: 表形式 ✓ / 比較軸3–5項目 ✓
