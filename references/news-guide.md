# News Guide（新闻事件类）

> news scene の字数管理は **見出し** と **本文** が独立した 2 階層。本文字数は scene specifications、見出し字数は NC-8 表参照。

## T1 Shared Constraints

**Register**: だ/である体統一。直接引用「」内のみ register 例外（NC-7 参照）。
**Dominant proof**: Logos — 事実・数値・出典のみ。Pathos完全排除。
**CTA**: 絶対禁止（防災警報のみ例外）。情報の受動的消費のみ。

| Constraint | Rule | Verification |
|-----------|------|-------------|
| NC-1 事実先出し | 5W（いつ/どこで/誰が/何を/どう）を冒頭1文に凝縮 | 「5W先出し: ✓」 |
| NC-2 数値完全保持 | 全数値・単位を原文から変更なく保持。丸め込み禁止 | 「数値保持: ✓」 |
| NC-3 主観ゼロ | NC-3.ext 禁止/許容語彙 3 階層表参照。forward-lookingは推測体 | NC-3 scan |
| NC-4 時間属性必須 | 時点明示を少なくとも1箇所。NC-4.ext で scene 別粒度を規定 | 「時点明示: ✓」 |
| NC-5 CTA禁止 | アクション動詞・行動示唆・読者への呼びかけ禁止 | 「CTA不在: ✓」 |
| NC-6 出典処理 | NC-6.ext 4 階層分類に従う。広く報道済みデータは免除（例: 日経平均終値） | NC-6 scan |
| NC-7 レジスタ純度 | だ/である体統一。直接引用「」内は原発話文体保持（下記参照） | 「文末: だ/である ✓」 |
| NC-8 見出し制約 | 字数 hard cap (下記表)、modality と `！？…` は hard 禁止、体言止めは soft 推奨 | 「見出し: X字 (≤Y) ✓」 |
| NC-9 分析パラ分離 (soft) | 分析パラグラフを末尾に物理隔離（Bloomberg 型、wrap/digest 推奨）。**Craft 層 D7 採点で評価。Step 5 強制 check には入れない** | — (soft) |

**Analysis cap**: 事実/引用 ≥ 70%、分析/見通し ≤ 30%。超過時は`trade-ideas`に再ルーティング。
State「分析比率: XX% (≤30% ✓)」in Step 5.

### NC-7 レジスタ純度（改訂版、ADR-007 §2.2 全文転記）

> だ/である体以外の文末禁止。RL-11 news 適用。同一記事内で常体／敬体の混在は原則不可。
>
> **例外**：直接引用「」内では被引話者の原発話文体を保持する。共同通信『記者ハンドブック』第 13 版 "文末統一" 節の業界慣行に従う。地の文（ナラタティブ）は依然としてだ/である体を維持しなければならない。
>
> | Context | Register 要求 |
> |---|---|
> | 地の文（本文・見出し・要約） | だ/である体、例外なし |
> | 直接引用「」内（発話引用） | 原発話を保持（です／ます／口語いずれも可） |
> | 間接引用（「〜と述べた」「〜と発表した」） | 地の文扱い — だ/である体 |
> | 注・補足括弧（）内 | 地の文扱い — だ/である体 |

`verify` は地の文文末スキャン時「」内を除外し、「」ペアリングを前置チェックする（ADR-007 §2.2）。

### NC-8 見出し制約 (ADR-009)

(a) 各 scene の見出し字数は下表 hard limit に従う。違反は Step 5 で fail。

| Scene | 見出し hard limit |
|---|---|
| news/flash | ≤15 字 |
| news/alert | ≤15 字 |
| news/wrap | ≤32 字 |
| news/earnings | ≤32 字 |
| news/digest | ≤32 字 |
| news/indicator | ≤24 字 |

(b) 体言止めは **soft 推奨**（Craft 層 D7 採点）。flash/alert/wrap/digest は実測 70-90% が体言止め支配的、earnings は動詞型（"○○、増益" / "△△、減配"）30-40% を許容、indicator は半構造テンプレ（"CPI、予想超え"）を許容。
(c) 見出しに modality marker（〜と見られる、〜の見方、〜の可能性）を含めない（hard、Step 5 fail）。
(d) 見出しに `！` `？` `…` を使用しない（hard、Step 5 fail）。

数値整合: 本表は SKILL.md RL-4 hard cap 表と同一値（RL-4 が定義、本表が運用ガイド）。

### NC-3.ext 主観ゼロ語彙 3 階層

| 階層 | 語彙例 | 扱い |
|---|---|---|
| **高（事実文・見通し文ともに禁止）** | 「思われる」「感じられる」「驚くべき」「衝撃的な」「素晴らしい」「間違いなく」「確実に」「〜となるだろう」 | 削除 or 数値置換 |
| **中（事実文 NG / 限定的に見通し文許容）** | 「〜ではないか」「〜だろうか」「楽観的な」「悲観的な」 | 「〜との指摘もある」に言い換え |
| **低（forward-looking 文に許容、事実文に混入 NG）** | 「〜とみられる」「〜の可能性がある」「〜との見方」「〜との観測」「〜が注目される」 | 推測体としてそのまま使用 |

事実文に低階層 hedging を配置すると Craft 層 D7 で減点（例: NG「決算を発表したとみられる」／OK「来期も成長するとみられる」）。

### NC-6.ext 出典 4 階層と引用タグ

| 階層 | 例 | 出典必要性 | 推奨タグ |
|---|---|---|---|
| **A. 公開指標** | 日経平均終値・ドル円・TOPIX | 不要（指数名で文脈成立） | — |
| **B. 公的統計** | CPI・雇用統計・鉱工業生産・GDP | 必須 | 「〇〇省が発表した」「〇〇省によると」 |
| **C. 企業開示** | 決算値・業績予想・配当・人事 | 必須 | 「〇〇社が発表した」「〇〇社の発表によると」「〇〇社が明らかにした」 |
| **D. 機関予想・サーベイ** | アナリスト予想・市場コンセンサス | 必須＋媒体名 | 「ブルームバーグが集計した市場予想によると」「QUICKコンセンサスでは」 |

引用タグ語彙: `〜が発表した` (第一選択) / `〜によると` (第二選択) / `〜が明らかにした` (会見) / `〜と述べた・〜との見解を示した` (発言間接引用) / `〜とコメントした` (口語寄り、決算社長コメントのみ) / `〜だという・〜とされる` (伝聞、多用注意)。

### NC-4.ext scene 別時点粒度

| Scene | 推奨粒度 | 例 |
|---|---|---|
| flash | 分単位 or 場区分 | 「14時35分」「大引け」「前場終了時点」 |
| alert | 分単位 | 「14時35分時点」 |
| wrap | 日付 + 「本日／〇日」 | 「29日の東京市場」 |
| earnings | 会計期間 | 「2026年3月期第2四半期」「通期予想」 |
| indicator | 対象月／四半期 | 「X月のCPI」「Q3 GDP」 |
| digest | 各トピックごとの対象時点 | トピック単位で粒度可変 |

## T2 Scene Specifications

### flash（市況速報）
**本文**: ≤140字 ／ **見出し**: ≤15 字
**情報構造**: 事実（数値）→ 影響（1文）
**Register**: だ/である体。見通し禁止（分析cap超過）
**Checkpoint**: 数値付き事実 ✓ / 140字以内 ✓ / CTA不在 ✓

### wrap（市場まとめ・ラップアップ）
**本文**: 300–700字 ／ **見出し**: ≤32 字
**情報構造**: 主要指標サマリー → 個別イベント列挙 → 明日の注目点（事実ベース）
**Register**: だ/である体。見通し文には必ず推測表現
**Checkpoint**: 主要3指標以上 ✓ / 明日の注目は推測体 ✓

### earnings（決算速報）
**本文**: 150–400字 ／ **見出し**: ≤32 字
**情報構造**: 決算概要（実績 vs 会社予想）→ 前年同期比 → 主要数値 → 会社コメント（引用）
**必須**: 実績・会社予想・前年同期比の 3 値セット（新規上場は 2 値可）。会社コメントは「」引用体（NC-7 例外領域）
**Checkpoint**: 3 値セット ✓ / 会社コメント引用 ✓

### indicator（経済指標速報）
**本文**: 100–300字 ／ **見出し**: ≤24 字
**情報構造**: 指標名 → 実績値 → 市場予想 → 前回値 → 市場への影響（事実ベース）
**必須**: 実績・予想・前回の三値セット
**Checkpoint**: 三値セット ✓ / 影響は事実ベース（断定しない）✓

### digest（ニュースダイジェスト）
**本文**: 500–1200字（複数トピック列挙）／ **見出し**: ≤32 字
**情報構造**: トピック見出し（1行）→ 事実サマリー（2–4文）の繰り返し
**必須**: トピック間は箇条書き/番号付きリスト
**Checkpoint**: 各トピック≤4文 ✓ / 見出し体言止め ✓

### alert（相場アラート）
**本文**: ≤100字 ／ **見出し**: ≤15 字
**情報構造**: 閾値到達事実 → 数値 → 時点
**例**: 「日経平均が40,000円を突破した（14時35分時点）。」
**Checkpoint**: 数値・時点両方 ✓ / 100字以内 ✓
