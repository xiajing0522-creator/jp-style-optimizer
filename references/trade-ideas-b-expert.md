# Trade-Ideas **Type B** Expert Module — かぶリッジ ベンチマーク

## Stance

Provides **frame templates** and **micro-pattern repertoire** for `--style trade-ideas` の **Type B**(Blog/SEO 個別株深掘)sub-form 専用,derived from 22 articles on `kabu.bridge-salon.jp`(かぶリッジ、株式会社インベストメントブリッジ運営、moomoo 証券 in-house SEO blog、2026 年運営)。

22 篇分布:個別株深掘 16 篇 / テーマ・ランキング 6 篇。長型 ≥10,000 chars 14 篇 / 中型 6,000–10,000 chars 8 篇。

### trade-ideas 内部の 2 sub-form ルーティング

trade-ideas style は **2 つの sub-form** に分離管理する。本ファイルは **Type B 専用**:

| Sub-form | 代表媒体 | レジスタ | 字数 | 主導 scene | 専用 reference |
|----------|---------|---------|-----|----------|---------------|
| Type R — Brokerage Research / Morning Brief | みんかぶ・株探・各社モーニングレポート | dual-register(分析=だ/である、読者=です/ます) | 80–500 字 | flash / earnings / morning(短) / picks | `trade-ideas-r-expert.md`(未作成、暫定で `trade-ideas-guide.md` を使用) |
| **Type B — Blog/SEO 個別株深掘 ※本ファイル** | かぶリッジ・トウシル・楽天/SBI 証券コラム | **です/ます体 全文一貫** | 6,000–16,000 字 | theme / community / picks 拡張 | **本ファイル** |

**観察**:かぶリッジ 22 篇全文で「だ/である」体は **0 件**(soft_register 22/22)。Type B は SEO で零售個人投資家にリーチするため、券商 research の重い文体を避け「親しみやすい です/ます」で統一する。

**含意(本ファイル適用時のみ)**:既存の `trade-ideas-guide.md` にある dual-register 制約(TI-7、TI-C P0 ゲート由来)は **Type R 専用**。本ファイルが lazy-load された時点で TI-C は **免除**し、代わりに「です/ます体一貫」を要求する。Type R 適用時(本ファイル非 load)は従来どおり TI-C を有効化。

### Type B 判定ヒント(Step 3 ルーティング時)

以下のいずれかが該当 → Type B → 本ファイル lazy-load:
- 字数 ≥ 5,000 字
- 個別銘柄を「深掘」する記事構造(基本情報 + 業績 + 比較 + 将来性)
- SEO 媒体(かぶリッジ・トウシル・楽天証券コラム・SBI 証券コラム 等)
- 個人投資家向けトーン(口語混在、「〜のかな?」「〜だね」)

該当しない(券商 research / 短型 morning brief / 速報) → Type R → `trade-ideas-guide.md` の dual-register 制約を使用、本ファイルは load しない。

## 2. Micro Patterns — Type B 文案特点(22 篇統計付き)

### 2.1 導入パターン(リード / フック)

| パターン | 命中 | 例 |
|---------|-----|---|
| **「かぶリッジの結論」box 先行型** | 20/22 | 第 1 段に💡アイコン + 結論 3〜4 行 bullet |
| **数値密度濃厚型** | 22/22 | リード 1,200 字以内に数値 ≥ 3 件(株価・倍率・期間) |
| **修辞疑問共感型(口語)** | 9/22 | 「〜だろう?」「〜のかな?」「〜だね…」 |
| **wh 列挙痛点型** | 12/22 | 「なぜ下がった?」「どこで買える?」「いつが買い時?」 |
| **キャラクター挿入型** | 5/22 | 「〜だワン!」「〜ワン」(犬キャラ「ちょる子」「ひねけん」等) |

**典型導入構造**(冒頭 800〜1,500 字、90% 以上の記事に共通):

1. **タイトル付近の主疑問** —「〇〇の株価はなぜ▲▲?」
2. **想定読者の状況描写**(口語の独白)—「〜って実際どうなの?」「〜って人も多いんじゃないかな」
3. **「💡 かぶリッジの結論」box** — bullet 3〜4 行で結論先取り
4. **本記事のスコープ宣言** —「本記事では〇〇を分かりやすく解説します」
5. **編集部・監修者署名** — 「執筆:かぶリッジ編集部」+ 「監修者:たけぞう(元証券ディーラー、約 30 年勤務)」プロフィール
6. **本記事の注意事項** —「投資を推奨・勧誘する目的はございません。情報の取捨選択や投資判断は各ユーザー様のご判断・責任にて」18/22 で明示

### 2.2 見出し措辞のレジスタ

**頻出フレーズ**(H1/H2 で繰返し):

| カテゴリ | 例 |
|---------|---|
| 主疑問提示 | 「なぜ▲▲?」「どうなる?」「買うべきか?」「危ない?」「やめとけ?」 |
| 理由分解 | 「3つの理由」「4つの特徴」「N 選」 |
| 将来予測 | 「10年後どうなる?」「今後の見通し」「今後の将来性」 |
| ファンダ整理 | 「事業内容・業績・株主還元」「基本情報」 |
| 比較軸 | 「競合 N 社を比較!」「強みと弱み」 |
| 結論再述 | 「【まとめ】〇〇は△△!」 100% 出現 |
| 専門家 | 「当社アナリストの見解」「専門家のポイント」 |
| 強調 | 「【テンバガー】」「【10倍株候補?】」「【2026年最新】」 |

**避けるべき見出し措辞**(本媒体では出現せず、または規制違反):

- 「絶対」「100%」「必勝」「儲かる」(投資推奨に該当する可能性)
- 動詞終止「〇〇を買おう」「〇〇に投資しよう」(推奨断定が強すぎる)
- 「〇〇を売れ」「〇〇から逃げろ」(売買命令調)

### 2.3 句法・修辞ツールキット

**A. 全文 です/ます体**(22/22):分析・見通し・基本情報を含む全段落で **です/ます体** 一貫。「〜と考えられます」「〜と言えるでしょう」「〜と推察されます」等の柔らかい推測語尾。

**B. 数値密度**:全 22 篇でリード 1,200 字以内に数値 ≥ 3 件(株価水準・倍率・期間)。本文中も「営業利益 +110%」「2026 年 4 月 16 日」「年初来高値 2,581 円」等具体数値が頻出。

**C. PER/PBR/ROE/配当利回り 等指標**:9/22 で言及。Type R(券商 research)と比べると低頻度だが、ファンダメンタル比較セクションでは必須に近い。

**D. 「💡 このパートの要約」内ハイライト型**:各 H2 セクションの開始部に「💡 このパートの要約」box を配置し、bullet 3 行で要点先取り。読者は scan で要点把握可能。

**E. 視覚分離マーカー**:
- 「💡」(結論・要約 box)
- 「※」(注釈、平均 5〜10 件/篇)
- 「【】」(強調マーカー、見出しに多用)
- 「▼」「●」(箇条書き先頭)
- 「TradingView より」「同社 HP より」(画像出典)

**F. 監修者・執筆者・編集部 3 層構造**(20/22 標準):
1. **執筆者**:「かぶリッジ編集部」(20 年以上の投資家向けサービス運営の自社ブランディング)
2. **監修者**:「たけぞう」(合同会社 Next Meeting 代表、元証券ディーラー約 30 年、最大 50 億円利益)等の実名 + 経歴
3. **専門家のポイント**:監修者本人の発言形式 box(3〜7/22)

**G. キャラクター挿入(任意)**:「ちょる子」「ひねけん」等の犬キャラが「〇〇だワン!」と発言するシーン(5/22)。専門用語の言い換え・読者目線の問いかけに使用。

### 2.4 肯定/否定の切換

**典型「下落・なぜ?」型 article 構造**:

| 順 | 内容 |
|----|------|
| 1 | 下落理由 3〜4 個の客観整理(否定面) |
| 2 | ファンダメンタル中立評価(業績・配当) |
| 3 | 将来性・反発材料(肯定面) |
| 4 | 「当社アナリスト見解」(中立判断) |
| 5 | 「【まとめ】判断は慎重に!」(両面提示で結論を読者に委ねる) |

**バランス比率**:否定要因 30〜40% / 肯定要因 30〜40% / 中立(ファンダ + 専門家)20〜30%。**完全な Buy/Sell 推介を避け、「判断材料の提供」に徹する**(投資推奨・勧誘の規制回避)。

### 2.5 数値・出典のスタイル

- **時点明示**:「2026 年 4 月 16 日」「2025 年 10 月時点」「2026 年 5 月最新」 — 18/22 で出現
- **チャート出典**:「TradingView より」9/22、「同社 HP より」「決算資料より」13/22
- **moomoo アプリ言及**:8/22(in-house ツール推し、特に「moomoo 証券アプリで分析」H3)
- **公的機関参照**:「金融庁」「日銀」 — 4/22(Type B では Type R より低頻度)

### 2.6 末尾の典型構造

| 順 | 内容 |
|----|------|
| 1 | 「当社アナリストの見解」(任意 4/22) |
| 2 | 「【まとめ】〇〇の主要論点 + 判断助言」H2 |
| 3 | 結論 bullet 3〜5 行(両面提示の再述) |
| 4 | 「判断は慎重に!」「ぜひ参考にしてください」型の柔軟結尾 |
| 5 | 関連記事リンク(銘柄紹介・IPO 情報) |

**CTA は「無料相談」「公式サイトで詳しく」型ではなく、「moomoo 証券アプリで分析」「メルマガ登録」型のソフト誘導**。「今すぐ買え」「絶対買うな」型は使わず、AC-12 / RL-12 と整合。

## 4. Anti-Patterns(避けるべき書き方)

| anti-pattern | 衝突 | 代替 |
|-------------|-----|------|
| 「絶対上がる」「100% 儲かる」「確実」 | RL-3 / AC-16(campaign 由来規制を準用) | 「期待できます」「実績があります」 |
| 「今すぐ買い時!」+「!」 | AC-12 流用 | 「現在の株価水準は注目に値します」 |
| 主観的否定「最悪」「使えない」「ゴミ」 | RL-3 | 「業績見通しに不安要素があります」 |
| Buy/Sell 命令調「買え」「売れ」「逃げろ」 | RL-12(news CTA 禁止に準ずる) | 「判断材料を提供します」「参考にしてください」 |
| 数値出典なし「業績 No.1」「業界最高水準」 | RL-9 / PC-3 | 「2025 年 12 月期 売上高 X 兆円(同社決算より)」 |
| 投資推奨明記なし(免責ブロック欠落) | (透明性) | 「投資を推奨・勧誘する目的ではありません」明記必須 |
| 分析段で「だ/である」混入(Type B では NG) | RL-11(本ファイル sub-form 制約) | です/ます体一貫 |

**重要**:Type B では本来 trade-ideas P0 の TI-C(dual-register)免除。代わりに「です/ます一貫」のチェックを実施。
