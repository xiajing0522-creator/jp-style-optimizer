# Product Expert Module — zuu / NET MONEY ベンチマーク

## Stance

Provides **frame templates** and **micro-pattern repertoire** for `--style product`, derived from 20 articles on `zuu.co.jp/media` (NET MONEY 編集部、2026 年運営)。case-library.md(brand voice signatures)とは異なり、本ファイルは**汎用の構造・修辞ベンチマーク**を提供する。

20 篇分布:FX 比較 5 / クレカ 比較 4 / 仮想通貨 1 / カードローン 1 / 不動産投資 1 / 株式・NISA 4 / インタビュー 3 / その他 1。長型(≥18,000 chars)11 篇 / 短型 9 篇。

## 1. Macro Framework — 3 つの記事タイプと章節テンプレート

### Type A — Compare-Long(比較ランキング型 / 33,000–70,000 chars)

**該当 product scene**:`compare`、`detail`(複数競合の徹底比較)

**章節テンプレート**(必須順):

| 順 | 章節タイトル例 | 役割 |
|----|---------------|------|
| 1 | 〇〇の選び方【重要な N 個のポイント】 | 判断軸の事前提示(独自の選定基準を編集部が定義) |
| 2 | 〇〇おすすめランキング N 選 / 比較ランキング | 主体ランキング(各候補に独立 H3 + 比較表) |
| 3 | 〇〇のメリット / デメリット | 双面披露(肯定と否定を別段落並列) |
| 4 | 【目的別】おすすめの〇〇 / 属性別で選ぶ | セグメント分岐(初心者 / 学生 / 高利用 / 低収入 等) |
| 5 | 〇〇の始め方 / 申込み手順 | STEP1〜STEP5 の操作フロー |
| 6 | 〇〇の審査基準 / 注意点 | リスク・前提条件の開示 |
| 7 | 〇〇に関するよくある質問 | FAQ(8〜15 件、具体的質問形式) |

**特徴**:章節 8〜13 個。H3 多用(各 candidate を独立 H3 化、典型 30〜60 個)。比較表で TOP3 ハイライト。各 H3 配下に 200〜400 字の解説 + 公式リンク。

### Type B — Single-Service Review(単一サービス評価型 / 6,000–17,000 chars)

**該当 product scene**:`detail`(単品紹介)、`onboard`

**章節テンプレート**:

| 順 | 章節タイトル例 | 役割 |
|----|---------------|------|
| 1 | 〇〇とは?基本情報と仕組み | サービス定義 + 運営会社 |
| 2 | 〇〇の特徴 / 強み / N つの理由 | 主要 USP 列挙(各 H3 で独立) |
| 3 | 〇〇のメリット | ベネフィット列挙(具体数値必須) |
| 4 | 〇〇のデメリット / 注意点 / 怪しいと言われる理由 | リスク開示(主観評判の客観整理含む) |
| 5 | 〇〇が向いている人 / 向いていない人 | マッチング判断(両セット必須) |
| 6 | 〇〇の始め方 / 申込みの流れ | STEP1〜STEP5 |
| 7 | 〇〇に関するよくある質問 | FAQ(5〜10 件) |
| 8 | まとめ | 結論再述 + CTA |

**特徴**:章節 6〜9 個。Type A より H3 少ない(典型 8〜30 個)。「向いている / 向いていない」両極化が定番。

### Type C — Interview(エキスパート対談型 / 7,500–9,000 chars)

**該当 product scene**:既存 product scene にはマッチせず、`detail` に B2B/思想性投影として応用可。または新 sub-scene `interview` 候補。

**章節テンプレート**:H1 = 経営者の挑戦 / 業界変革のキャッチコピー。H2 = 物語段落(6〜8 個)。

| 順 | 段落の役割 |
|----|----------|
| 1 | 業界課題と挑戦の宣言(「〇〇の業界に風穴を開ける」) |
| 2 | 過去の障壁・苦労(「ライセンス取得の壁」「銀行時代の大失敗」) |
| 3 | 解決アプローチ(共創 / DX / 顧客目線) |
| 4 | 内側の哲学(「公人としての誠実さ」「文化の融合」) |
| 5 | エコシステム / プラットフォーム構想 |
| 6 | 成功事例 / 数値証拠 |
| 7 | 未来ビジョン(「2032 年 1,000 名体制」「社会のインフラへ」) |

**特徴**:H3 ほぼ無し(全文ストーリー駆動)。Q&A 形式の対談スタイル。**文体は product 主流から逸脱**(深層的・抽象的・ビジョン語彙多用)。一般的 product 文案執筆時には参考にしない、interview 専用。

## 2. Micro Patterns — 文案特点(20 篇統計付き)

### 2.1 導入パターン(リード / フック)

| パターン | 命中 | 例 |
|---------|-----|---|
| **修辞疑問共感型** | 18/20 | 「〇〇したい方が多いのではないでしょうか」「悩みを持つ方は多い」 |
| **複数 wh 列挙痛点型** | 16/20 | 「どこで口座開設したらよいのか」「どうやって選べばいいのか」「いくらから始められるのか」 |
| **結論先行型** | 6/20 | 「結論から述べると、〇〇は△△です」(根拠は後段で展開) |
| **数値+権威型** | 17/20 | 「世界シェア 1 位」「累計 15 万人以上」「2026 年 5 月時点」 |

**典型導入構造**(冒頭 200〜400 字、80% の記事に共通):
1. 想定読者の状況描写(「〇〇したいけど〜」)
2. 痛点の修辞疑問化(「〜と感じる方は多いのではないでしょうか」)
3. 本記事のスコープ宣言(「本記事では〇〇を解説します」)
4. 結論サマリー(「結論から述べると、〇〇です」)or 「この記事でわかること」bullet 3〜5 項
5. 著者・監修者署名(実名 + 資格 + 経歴)

### 2.2 見出し措辞のレジスタ

**頻出フレーズ**(H1/H2 で繰返し):

| カテゴリ | 例 |
|---------|---|
| 比較 | 「徹底比較」「N 選」「ランキング」「TOP N」「おすすめ N 社」 |
| 解説 | 「とは?」「仕組みをわかりやすく」「徹底解説」「3 つの要素」 |
| 双面 | 「メリット / デメリット」「向いている人 / 向いていない人」「評判の真相」 |
| 操作 | 「始め方」「選び方」「手順」「STEP1〜STEP N」「N 個のポイント」 |
| 強調 | 「【2026 年最新】」「【番外編】」「【目的別】」「【初心者必見】」「【100 人調査】」 |
| 不安解消 | 「失敗しない」「後悔しない」「怪しい?」「安全性を検証」「やめとけの真相」 |

**避けるべき見出し措辞**(本媒体では出現せず):
- 動詞終止「〇〇した」「〇〇する」型(ニュース文体に偏る)
- 単純名詞のみ「クレジットカード」「FX」(情報量不足)
- 「絶対」「必勝」「100%」(既存 RL の AC-16 / CMP-2 違反)

### 2.3 句法・修辞ツールキット

**A. 体言止め交ぜ**:見出し階層は体言止め優位(「〇〇とは?」「メリット」「始め方」)。本文は**です/ます体**で安定(混用なし)。

**B. 数値密度**:全 20 篇で平均 163 件/篇の数値出現。「〇円」「〇%」「〇選」「〇万人」「〇通貨ペア」 ≧ 1 件 / 200 字。汎用形容詞「お得」「便利」のみで終わる段落は皆無。

**C. 「あなたに合った〜」型決断橋渡し**:
- 「自分に合った〇〇を見つけよう」
- 「ぜひ参考にしてください」
- 「あなたにぴったりの 1 枚を見つけてください」
- 「ライフスタイル別」「目的別」「属性別」

**D. 「結論」「ポイント」「特徴」段落分割**:長段落は「ポイント:」「【結論】」「■〇〇の基本情報」等のラベル付き mini-block で分割。読者は scan で要点把握可能。

**E. 視覚分離マーカー**:
- 「※」(注釈、平均 60 件/篇)
- 「＼〇〇／」(キャンペーン強調)
- 「■〇〇」(ヘッダー強化)
- 「PR」「【PR】」(広告開示の明示、affiliate 透明性)

**F. 専門家署名 3 層構造**(detail / compare scene でほぼ標準):
1. **監修者**:実名 + 資格(CFP / FP 技能士 / 証券アナリスト 等)+ 業歴 数十文字
2. **著者**:実名 + 経歴 + 専門領域
3. **編集者** or **校正会社**:法人名 + 役割

### 2.4 肯定/否定の切換

**肯定ブロック**:メリット / 強み / 魅力 / おすすめポイント
**否定ブロック**:デメリット / 注意点 / 後悔しないために / 落ちる原因

**比率の慣習**:メリット 55〜65% / デメリット 35〜45%(完全均衡ではないが、否定側を 30% 未満に圧縮しない)。

**否定の措辞は具体的かつ建設的**:
- ✓ 「営業電話がしつこい」「キャンペーンの条件が厳しい」
- ✗ 「ダメ」「最悪」「使えない」(主観的すぎる)

### 2.5 数値・出典のスタイル

- **時点明示**:「2026 年 3 月時点」「2025 年 9 月時点」「2026 年 5 月最新」 — 17/20 で出現
- **公的機関参照**:「金融庁」「日銀」「国税庁」 — 7/20 で参照リンクあり
- **大手メディア参照**:「Bloomberg」「Reuters」「日経」 — 3/20
- **独自調査**:「【100 人調査】」「【NET MONEY 編集部独自調査】」 — クレカ系で頻出

### 2.6 末尾の典型構造

| 順 | 内容 |
|----|------|
| 1 | 「まとめ」見出し |
| 2 | 記事の主要結論 3〜5 行(箇条書きまたは段落) |
| 3 | 想定読者別の最終推薦(「〇〇したい方には△△」) |
| 4 | 公式サイトリンク(複数候補に対し並列) |
| 5 | 「よくある質問」(末尾配置 or 独立章節) |

CTA は「申込みはこちら」「公式サイトを見る」「無料で始める」型。「今すぐ」「絶対」型は使わず(既存 AC-12 / AC-16 と整合)。

## 3. Scene 別適用ガイド

| product scene | 主推荐タイプ | 章節数目安 | 必須要素 |
|---------------|------------|-----------|---------|
| `short`(USP カード) | — | 1〜2 段落のみ | 数値ベネフィット 1 件 + Trial CTA |
| `detail`(詳細説明) | Type B | 6〜8 章 | 監修者署名 / メリット・デメリット並列 / FAQ |
| `manual`(手順書) | Type B 簡略 | 4〜5 章 | STEP1〜N / 各ステップに UI 要素名 |
| `script`(動画台本) | — | 3〜5 章 | 数値フック導入 / 1 文 ≤30 字 |
| `release`(更新情報) | — | 3〜4 章 | 時点明示 / 影響範囲 / 手順 |
| `onboard`(入会) | Type B | 5〜6 章 | Reading-Outcome Preview / STEP / FAQ |
| `compare`(比較) | Type A | 8〜13 章 | 選び方軸 / ランキング / 目的別 / FAQ |

## 4. Anti-Patterns(避けるべき書き方)

zuu サンプルでも稀に出現するが、本 skill の RL/AC と衝突するもの:

| anti-pattern | 衝突 | 代替 |
|-------------|-----|------|
| 「絶対」「必勝」「100%」 | AC-16 / CMP-2 | 「期待できる」「実績がある」 |
| 「今すぐ」+「！」 | AC-12 | 「無料で始める」「気軽にお試し」 |
| 主観的否定「最悪」「使えない」 | RL-3 | 「〇〇な点は注意が必要」 |
| 「〜とみられる」「〜見込み」(キャンペーン投影) | RL-11 (3) | product では断定 or 数値 |
| 数値出典なし「業界最速」 | RL-9 / PC-3 | 「業界最低水準*」+ 注記で時点・条件明示 |
| affiliate 開示なし | (透明性) | 「PR」「【PR】」「広告」マーク必須 |

## 5. 引用例(代表的段落の抜粋、参考用)

### 例 5.1 — 修辞疑問共感型導入(複数 wh 列挙)

> 仮想通貨を始めてみたいけど会社が多すぎて、どこで口座開設したらよいのかわからない、取引画面やツールが使いやすいサービスを知りたい、セキュリティ対策がしっかりしている会社で口座開設したい——仮想通貨取引をこれから始めようとしている初心者の人の中には、どのような軸で仮想通貨取引所を選ぶべきかわからず、「とりあえずおすすめを知りたい」と思うケースも多いのではないでしょうか。

(`btc-exchange`、出典:zuu.co.jp/media/cryptocurrency/bitcoin-exchange)

### 例 5.2 — 結論先行 + 双面披露

> 結論から述べると、オルタナバンクは第一種金融商品取引業者として金融庁の監督下で運営されており、正式な運用実績においても元本毀損率 0% を維持しているサービスです。本記事では、2026 年 3 月時点の最新情報をもとに、オルタナバンクが「怪しい」と言われる理由を整理したうえで、安全性やメリット・デメリットをバランスよく解説します。

(`alternabank`、出典:zuu.co.jp/media/stock/alternabank-reputation)

### 例 5.3 — 「向いている / 向いていない」 マッチング型結論

> RENOSY の利用が向いている人:長期的視点で資産形成したい / 本業が忙しく手間を掛けたくない / 都市部の堅実な物件に投資したい
> RENOSY の利用が向いていない人:短期で大きなリターンを狙いたい / 自分で物件を選びたい / 地方物件に投資したい

(`renosy`、出典:zuu.co.jp/media/real-estate-investment/renosy-review)

### 例 5.4 — STEP フロー(操作手順)

> STEP1:公式サイトから資料請求・無料相談を申し込む
> STEP2:担当エージェントとオンラインで個別面談
> STEP3:希望に沿った物件の提案・収支シミュレーション
> STEP4:売買契約の締結と金融機関へのローン申し込み
> STEP5:物件の引き渡しとオーナーアプリでの運用開始

(`renosy`)

### 例 5.5 — 専門家署名 3 層構造

> 監修者:トータルマネーコンサルタント 新井 智美(あらい・ともみ)— CFP / 1 級ファイナンシャルプランニング技能士 / DC プランナー / 住宅ローンアドバイザー / 証券外務員
> 著者:金融メディア NET MONEY 編集部 — 2006 年創刊、月刊投資情報誌「NET MONEY」の Web 版を運営
> 校正・編集:株式会社ヴェリタ — 派遣元事業者 / 有料職業紹介事業者の資格を取得

(`visa-creditcard`)

## 7. 使用方法

本ファイルは Step 4 transform 段階で `--style product` ルーティング後に lazy-load される。`product-guide.md` と併用:

- `product-guide.md` — T1 + T2 制約(scene 別句長、F→B、CTA レベル)
- `product-expert.md` — **構造・修辞ベンチマーク**(本ファイル)
- `case-library.md` — ブランド固有の voice signature(任意)

**典型ワークフロー**:
1. Step 3 ルーティング → `--style product --scene compare`
2. Step 4 Layer 1(Structure)で本ファイルの **Type A 章節テンプレート**を参照
3. Step 4 Layer 2(Language)で 2.2 見出し措辞 + 2.3 句法ツールキットを適用
4. Step 6 Naturalness Check の Proxy 3(Reference comparison)で本ファイルの引用例 5.1〜5.5 と比較
