# Campaign Guide（营销活动类）

> **Routing note (ADR-005)**: campaign は dual-register 例外なし・全シーン です/ます 単一。体言止め主体のブランディング短文（情緒訴求・期限/特典欠如）が入力された場合は `--style marketing` への切替を検討する。

## T1 Shared Constraints

**Register**: です/ます体。Push タイトル・バナーは体言止め許容。
**Dominant proof**: Pathos (緊迫感・得する感情)。Logos は条件透明性のみ。
**CTA**: 必須。段階別: Cold→「まずは詳しくみる」/ Warm→「参加する」/ Hot→「今すぐ申し込む」。

| Constraint | Rule | Verification |
|-----------|------|-------------|
| CA-1 期限明示 | 具体的な終了日時を必ず記載（「〜月〜日まで」）。「期間限定」のみ禁止 | 「期限: [日付]確認: ✓」 |
| CA-2 条件透明性 | 特典獲得条件を本文レベルで記載。注釈のみ禁止 | 「条件明記: ✓」 |
| CA-3 特典具体化 | 特典名称を具体的に記載。「素敵なプレゼント」禁止 | 「特典名: [具体名]確認: ✓」 |
| CA-4 金額表現 | 金額に「相当」付加。「100%当選」禁止→「必ずもらえる」 | CA-4 scan |
| CA-5 CTA stage | CTA温度をターゲット段階に整合 | 「CTA stage: [Cold/Warm/Hot] ✓」 |
| CA-6 lifecycle一貫 | 同一キャンペーンのlaunch→remind→lastday→resultで表記統一 | CA-6 check (複数文案時) |
| CA-7 multi-state整合 | 参加前/達成/未達成で同一レジスタ・トーン | CA-7 check |
| CA-8 煽り適正化 | 緊迫感は事実ベース（「残りX日」「先着X名」）。根拠なき煽り禁止 | CA-8 scan |
| CA-9 リスク開示分担 | LP/EDM 末尾にリスク文言必須、Push/SNS/Banner は `*詳細はLP参照` 誘導必須。欠如は hard-fail (金商法37条) | 「リスク開示: [本文/参照] ✓」 |
| CA-10 特典タイプ語彙制限 | 報酬性主張は「割引/無料/キャッシュバック/ポイント付与」のみ安全。「お得」「特別」は単独不可・特典名併記必須。違反 soft-fail | CA-10 scan |
| CA-11 禁止語彙 | 下表 5 語を 0 件 scan。1 件で hard-fail (AC-16 連動) | 「CA-11 禁止語: 0 件 ✓」 |
| CA-12 景表法 抽選/総付 排他 | 「抽選/先着/もれなく」を同一特典内で 2 つ以上混用禁止。下表で付与条件→必須開示判定。混用 hard-fail | 「CA-12 付与条件: [抽選/先着/総付] ✓」 |
| CA-14 先頭意味完結 (Craft) | Push 本文 先頭 30 字、EDM 件名 先頭 15 字で特典額/期限/アクション結論先行。違反は craft 減点のみ | 「先頭完結: ✓」(craft) |
| CA-15 「期間限定」単独禁止 | 「期間限定/今だけ/期間中」を具体日時併記なしで使用禁止。下記 regex 検出で hard-fail (CA-1 強化, 景表法運用基準) | 「CA-15 期限併記: ✓」 |

### CA-11 禁止語彙テーブル（AC-16 同一・5 語固定）

| 禁止語 | 修正ヒント |
|---|---|
| 必勝 | 「実践的」or 削除 |
| 絶対 | 「自信を持って」or 削除 |
| 確実 | 「着実」「明確」 |
| 100% (100％) | 「もれなく」(総付のみ) |
| 間違いなし | 「裏付けある」or 削除 |

出典: 金商法37条 / JSDA 規則 / 景表法5条。

### CA-12 景表法 付与条件マトリクス

| 付与条件 | 分類 | 必須開示項目 | 併用禁止 |
|---|---|---|---|
| 抽選 | 一般懸賞 | 当選人数・抽選方法・発表・景品上限額 | もれなく/先着 |
| 先着 | 一般懸賞 | 先着人数・カウント基準・期限 | もれなく/抽選 |
| もれなく/全員 | 総付景品 | 取引条件・上限額（価額20% or 10万円の低い方） | 抽選/先着 |
| ポイント還元 | 総付景品 | 還元率・上限・付与時期・有効期限 | 抽選 |

**判定**: 同一特典内に 2 カテゴリ以上併存で hard-fail。例「抽選で先着50名様にもれなくプレゼント」は 3 カテゴリ混用 reject。

### CA-15 「期間限定」単独使用 検出 regex 例

```
/(?<!(?:[0-9０-９]{1,4}[年月日時分]|まで|〜))(期間限定|今だけ|期間中)(?![0-9０-９〜~・(（])/

# OK: 「3月31日まで期間限定」「期間限定 (〜4/30)」
# NG: 「期間限定キャンペーン実施中！」「今だけのチャンス」
```

修正: 具体日時併記 or 該当語削除し日付単独表記。

## T2 Scene Specifications

### launch（キャンペーン開始告知）
**字数**: LP 300–800字; Push≤80字; SNS≤140字
**情報構造**: 特典名称 → 条件 → 期限 → CTA
**Push形式**: タイトル≤20字（数字/特典名先出し）+ 本文≤60字
**SNS**: 絵文字許容（🎉📢）; キャンペーン名は【】括弧
**Checkpoint**: 特典名具体 ✓ / 期限明示 ✓ / CTA Hot or Warm ✓

### remind（リマインド・中間アラート）
**字数**: Push≤80字; EDM 200–400字
**情報構造**: 残り期間 → 現在状況 → 行動促進 → CTA
**Emphasis**: 「残りX日」「あとXステップ」等の進捗表現
**Checkpoint**: 残り期間明示 ✓ / CTA温度 = Warm or Hot ✓

### lastday（最終日・ラストチャンス）
**字数**: Push≤80字; SNS≤140字
**情報構造**: 「本日最終」→ 特典 → CTA（最ホット）
**Tone**: 最大の緊迫感。「今日で終了」「本日限り」必須
**Checkpoint**: 「本日」「本日最終」等の最終日マーカー ✓ / CTA Hot ✓

### result（結果通知・達成報告）
**字数**: Push≤80字; アプリ内 100–200字
**情報構造**: 結果（達成/未達成） → 特典内容 → 次のステップ
**達成トーン**: 「おめでとうございます！」許容（RL-11例外: 結果通知のみ）
**未達成トーン**: 「惜しい！次回も挑戦してください」
**Checkpoint**: 結果明記 ✓ / 次のアクション明示 ✓

### seasonal（季節/イベント連動型）
**字数**: LP 400–1000字; 全チャネル展開
**情報構造**: 市場コンテクスト → 特典 → 条件 → 期限 → CTA
**注意**: 市場コンテクストは導入のみ（≤20%）。主目的は特典訴求。
**Checkpoint**: 市場文脈≤20% ✓ / 特典内容が主体 ✓

### referral（紹介・友達招待型）
**字数**: SNS/EDM 200–400字
**情報構造**: 紹介特典（双方） → 条件 → 紹介方法 → CTA
**必須**: 紹介者特典と被紹介者特典を両方明示
**Checkpoint**: 双方向特典明示 ✓ / 紹介方法具体 ✓
