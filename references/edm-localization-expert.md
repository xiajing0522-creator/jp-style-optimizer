# EDM Localization Expert(post-DS editorial pass for trade-ideas via EDM channel)

## 0. Scope & Application

このリファレンスは、**LLM(DS 3.2 等)出力の日本語コピーをネイティブ編集者が EDM 配信形態に整える際の編集パターン集**である。274 ペアの実証 corpus(2 年分・95 週次バッチ)から抽出した 14 系統 pattern + silent baseline を収録。

### 適用条件
- **入力が「LLM/機械翻訳の日本語出力(初稿)」**である場合(== 完全な人間原稿でない場合)に Layer 2 後段で実行
- `--style campaign --scene edm` ルート + 内容が trade-ideas / 投資情報系の場合(== Trade Idea EDM)
- `--style trade-ideas` で配信媒体が EDM(件名/プリヘッダー/Title/Content/CTA 構成)の場合

### 適用しない
- 完全な人間原稿(LLM 通過なし)
- compliance / disclosure / disclaimer(別ガイド適用)
- 短型 SNS / Push のみ(EDM 構造を持たない場合)

### Step 4 内での実行位置
Step 4 Layer 2(language)の **末尾**、Layer 3(proof & frame)の前。pattern を順次 scan し、検出時に LOC 形へ置換。Silent baseline は触らない。

## 1. Pattern Catalog(14 系統)

### A. Brand / Entity naming

#### A1. `moomoo` → `moomoo証券`(法人正式名化)【頻出】
- ルール: 単独「moomoo」が会社/サービス主体の文脈で出現する場合、`moomoo証券` に置換。「moomooアプリ」も `moomoo証券アプリ` 化が定型。
- 例:
  - DS: `moomooと一緒に投資のヒントを探りましょう！` → LOC: `moomoo証券と一緒に投資のヒントを探りましょう！`
  - DS: `moomooアプリをダウンロードする【iOS/Android】` → LOC: `moomoo証券アプリをダウンロードする【iOS/Android】`
  - DS: `moomooをご利用の皆様へ` → LOC: `moomoo証券をご利用の皆様へ`
- 例外: 商標表記としての「moomoo Securities」やコミュニティ機能名「moomoo コミュニティ」等のサービス固有名は touched せず。

#### A2. セクター/テーマ裸名詞 → `◯◯株 / ◯◯関連株 / ◯◯銘柄` 補完【頻出】
- ルール: 「買いの対象」「上昇期待」等の文脈で裸の業種/テーマ名が出る場合、`株`/`関連株`/`銘柄` を補い指示対象を明示化。
- 例:
  - DS: `銀行や金属に再び買いの波？` → LOC: `銀行や金属株に再び買いの波？`
  - DS: `半導体の復活は本物か？` → LOC: `半導体関連株の復活は本物か？`
  - DS: `いま最も勢いのある小売4選！` → LOC: `いま最も勢いのある小売り銘柄4選！`
- 逆方向あり: `「空調・電設」厳選6銘柄` → `空調・電設厳選6銘柄`(括弧外し)

### B. Numeric formatting

#### B1. 全角数字 → 半角数字【頻出】
- ルール: タイトル/本文の全角アラビア数字 `７/３/５` 等は半角 `7/3/5` に統一。
- 例:
  - DS: `国内半導体７銘柄` → LOC: `国内半導体7銘柄`
  - DS: `エヌビディア決算で５％以上急騰！` → LOC: `エヌビディア決算で5％以上急騰！`
  - DS: `9月配当の３銘柄は？` → LOC: `9月配当の3銘柄は？`

#### B2. 4桁以上の千位カンマ挿入【中頻度】
- ルール: 円金額の「4万3000円」型は `4万3,000円` または `50,500円` 型へカンマ挿入。Title/強調行で適用、本文では揺れあり。
- 例:
  - DS: `日経平均、一時4万3000円下回る` → LOC: `日経平均、一時4万3,000円割れ`
  - DS: `5万円500円前後でを推移` → LOC: `50,500円前後で推移`
  - DS: `約2800万人の来場が見込まれる` → LOC: `約2,800万人の来場が見込まれる`

#### B3. `%` の全半角は文書テンプレートに従う【中頻度・揺れあり】
- ルール: corpus 内では `5%` と `5％` の両方が出現。テンプレート内一貫性が優先(同記事内では揃える)。Subject/Title では `％` 全角寄りが多いが規則ではない。
- 例:
  - DS: `エヌビディア決算で５％以上急騰！` → LOC: `エヌビディア決算で5％以上急騰！`(全角→数字半角・%は全角維持)
  - DS: `中東の和平期待で日経平均3%急反発！` → LOC: 同上(半角 % 維持)

### C. Verbal phrasing(金融用語化)

#### C1. 「下回る」→「割れ」(節目下抜け)【散見・n=2、注意】
- ルール: 株価が節目を下抜く際の説明動詞は、相場慣用句「割れ」(体言止め)へ置換。
- 例:
  - DS: `日経平均、一時4万3000円下回る` → LOC: `日経平均、一時4万3,000円割れ`
- 注: 観察 n=2、Subject/Title 限定。本文で同様の置換が一貫しているとは確認できず、過剰適用注意。

#### C2. カタカナ口語動詞 → 漢語化【散見】
- ルール: 「アップ」「上がる」等のビジネス文として軽い動詞を「上昇」へ。
- 例:
  - DS: `安定政権誕生なら株価アップか?` → LOC: `安定政権誕生なら株価上昇か？`
  - DS: `半導体・AI関連株は今後も上場期待？` → LOC: `半導体・AI関連株は今後も上昇期待？`(DS の「上場」typo の修復)

#### C3. 強い断定語 → 中庸語に弱化【頻出】
- ルール: `確約 / 復活！ / 再開へ！` 等の断定/強調を `発表 / 再浮上か？ / 再開へ？` 等へ軟化。煽り抑制と RL-3 主観排除に整合。
- 例:
  - DS: `1000億円超の自社株買い確約` → LOC: `1000億円超の自社株買い発表`
  - DS: `「高市トレード」復活！` → LOC: `「高市トレード」再浮上か？`
  - DS: `日銀、12月利上げ再開へ！` → LOC: `日銀、12月利上げ再開へ？`

#### C4. 「て+る」省略形 → 標準書き言葉「ている」【散見・n≈3】
- ルール: 口語省略「買ってる/負けてる」等は editor が必ず「買っている」型に展開。
- 例:
  - DS: `大口投資家が買ってる銘柄は？` → LOC: `大口投資家が買っている銘柄は？`

---

### D. Particle / connective adjustment

#### D1. 「年初来で」→「年初来から」【n=1・注意】
- ルール: 期間起点の助詞 `で` を `から` に置換。
- 例:
  - DS: `年初来で株価上昇した大型高配当株は？` → LOC: `年初来から株価上昇した大型高配当株は？`
- 注: 観察 n=1。一般化前に追加 sample 必要。

#### D2. 逆接の補強(連用「て」→「ても」)【散見】
- 例:
  - DS: `業績で勝って株価で負けてる!?` → LOC: `業績で勝っても株価で負けてる!?`

#### D3. 連体修飾 → 属格構造解きほぐし(「名詞+名詞」→「名詞の名詞」)【中頻度】
- ルール: 連結が密で読みづらい複合名詞を「の」で分節し、コピュラ的属格化。
- 例:
  - DS: `AI成長性と不動産安定性を併せ持つ` → LOC: `AIの成長性と不動産の安定性を併せ持つ`
  - DS: `小林製薬は大反発するか？` → LOC: `小林製薬の株価は大反発するか？`(主語補足)

---

### E. Punctuation & spacing

#### E1. `?!` ⇄ `!?` 揺れ調整【散見・n=2、注意】
- ルール: 観察事例少。`日経平均、一時反落で買い水準?！` → `…買い水準!?` のように `!?` 寄せが見られるが、逆方向(`？！` 維持)も存在。明確な統一規則ではない。

#### E2. 半角 `?` → 全角 `？`(主見出し)【頻出】
- 例:
  - DS: `日本株は「押し目」買い水準?` → LOC: `日本株は「押し目」買い水準？`
  - DS: `安定政権誕生なら株価アップか?` → LOC: `安定政権誕生なら株価上昇か？`

#### E3. 読点/スペース挿入による息継ぎ整形【中頻度】
- ルール: 連続短文・列挙を半角スペース or 全角スペース or 読点で区切り、視認性を上げる。
- 例:
  - DS: `生成AI拡大思惑で電力株が急騰！？まだ上がる銘柄は？アナリストが続々目標株価引き上げ！` → LOC: `生成AI 拡大思惑で電力株が急騰！？まだ上がる銘柄は？ アナリストが続々、目標株価引き上げ！`
  - DS: `AIデータセンター投資急加速！` → LOC: `AIデータセンター投資、急加速！`
  - DS: `業績で勝っても株価で負けてる!?株主還元の充実で…` → LOC: `業績で勝っても株価で負けてる!?　株主還元の充実で…`(全角スペース)

#### E4. 主要見出しでの読点削除【散見】
- 例:
  - DS: `金利高で地銀株急騰、「経営統合」がブースターに？！` → LOC: `金利高で地銀株急騰「経営統合」がブースターに？！`

### F. Quote / bracket policies

#### F1. 新概念・限定語の鉤括弧化「」【頻出】
- ルール: 新規テーマ・固有概念の初出は「」で囲み固有性を示す。
- 例:
  - DS: `日本が先行！ダイヤモンド半導体関連銘柄は？` → LOC: `日本が先行！「ダイヤモンド半導体」関連銘柄は？`
  - DS: `テクニカル分析専門の公式アカウントが誕生しました！` → LOC: `「テクニカル分析専門」の公式アカウントが誕生しました。`

#### F2. ストレートクオート `"` → カーリー `“ ”`【頻出】
- ルール: ASCII 直線クオートは日本語タイポグラフィ用 curly quote(U+201C/U+201D)に倒す。
- 例:
  - DS: `”大化け”期待の量子コンピュータ関連23選！` → LOC: `“大化け”期待の量子コンピュータ関連23選！`
  - DS: `8月暴落は”かすり傷”？` → LOC: `8月暴落は“かすり傷”？`

#### F3. 既出 `「」` を外す/curly quote へ相互変換【散見】
- 例:
  - DS: `「空調・電設」厳選6銘柄` → LOC: `空調・電設厳選6銘柄`(タイトル短縮)
  - DS: `円高・利上げ影響は”軽微”と各社強気！` → LOC: `円高・利上げ影響は「軽微」と各社強気！`

---

### G. Technical term refinement

#### G1. `AI` → `生成AI`(LLM/Diffusion 文脈の specifier 補強)【頻出】
- ルール: 文脈が LLM/Diffusion を指す場合、汎称「AI」を「生成AI」に置換し技術範囲を狭める。
- 例:
  - DS: `AI時代の「サイバーセキュリティ」銘柄をチェック！` → LOC: `生成AI時代の「サイバーセキュリティ」銘柄をチェック！`
  - DS: `最強AI半導体株は？` → LOC: `最強の生成AI半導体株は？`
  - DS: `AI成長性と不動産安定性を併せ持つ` → LOC: `生成AIの成長性と不動産関連の安定性を併せ持つ`
- 例外: 固有名詞化された「AI」(`OpenAI` `xAI` `AI Overviews` 等)はそのまま。

#### G2. 一般語 ⇄ 業界用語 の双方向調整【中頻度】
- ルール: 読者リテラシーに合わせ語彙レベルを上下調整。一律の方向はない。
- 例(上方向):
  - DS: `期待値の高い銘柄を見逃さないために` → LOC: `有望な銘柄を見逃さないよう`
  - DS: `黒字転換続々で高成長に期待` → LOC: `黒転で高成長に期待`(短縮業界形)
- 例(下方向):
  - DS: `予想した社や、…社もあり` → LOC: `予想した会社や、…会社もあり`

### H. Ticker / 銘柄コード conventions

#### H1. 銘柄コード `<XXXX>` `(XXXX)` `<XXXX.JP>` は維持【silent baseline】
- ルール: editor は ticker 表記を基本変更しない。固有名詞の略称化(`日本電気` → `NEC`)は個別判断。
- 例:
  - DS: `三菱UFJフィナンシャル·グループ<8306>と三井住友フィナンシャルグループ<8316>` → LOC: 維持
  - DS: `アステリア(3853)` → LOC: 維持
  - DS: `日本電気や日立など国内大手` → LOC: `NECや日立など国内大手`(略称化、これは判断ベース)

### I. Sentence-final softening / hardening

#### I1. 末尾動詞・助動詞補完(体言止め回避)【頻出】
- ルール: DS の体言止め/短文/疑問形を、終止形・助動詞・動詞補完で締める。3 タイプ:
  - **(a) 助動詞型** — `…だ / …だろう / …が焦点だ / …のだろうか` 等で結ぶ。
  - **(b) 推量・期待型** — `…期待できよう / …見込まれる / …公算が大きい` 等で結ぶ。
  - **(c) 動詞補完型** — Title/見出しの体言止めに動詞を補い完成文化する(例:`…利益確保も` → `…利益確保も狙える`)。
- Content / unspec セクションで(a)(b)型が毎回発生、Title セクションで(c)型が散見。
- 例:
  - (a) DS: `15期連続増配を示せるかどうか。` → LOC: `15期連続増配を示せるかが焦点だ。`
  - (a) DS: `今回の決算が株価上昇のきっかけとなるか、それとも新たな下落の引き金となるか？` → LOC: `今回の決算が株価上昇のきっかけとなるのだろうか。`
  - (b) DS: `若干の為替差益が期待できる。` → LOC: `若干の為替差益が期待できよう。`
  - (c) DS: `「売り時」にもう迷わない!上昇トレンドを追いながら、急落時の利益確保も` → LOC: `「売り時」にもう迷わない!上昇トレンドを追いながら、急落時の利益確保も狙える`(2026-03-18 r28 / 2026-04-21 r14 で reproduce)

#### I2. 過剰煽り抑制(感嘆 → 疑問)【中頻度】
- 例:
  - DS: `“火柱”上げる暗号資産関連6銘柄！` → LOC: `“火柱”上げる暗号資産関連6銘柄？`
  - DS: `急騰間近の日本株が動き出すぞ！` → LOC: `急騰間近の日本株が動き出す？！`
  - DS: `5万円台に乗せる可能性も否定はできない!?` → LOC: `5万円台に乗せる可能性も否めない。`

#### I3. コミュニティ系見出しでの敬称口語化(「皆さま」→「みんな」)【散見・n=2、コミュニティ限定】
- ルール: コミュニティ投稿系の Title/CTA に限り、敬称を口語化し親近感を上げる。EDM 一般本文では適用しない。
- 例:
  - DS: `皆さんの見解をチェックしよう` → LOC: `みんなの見解をチェックしよう`

### J. 体裁(layout)

#### J1. 1行 → 2行 改行挿入(タイトル・サブタイトル分離)【中頻度】
- ルール: 主+副の構造を持つ 1 文 Title は、副題部を改行で物理分離。
- 例:
  - DS: `価格の節目を狙う！逆指値・トリガー注文の活用法` → LOC: `価格の節目を狙う！\n逆指値・トリガー注文の活用法`
  - DS: `新機能リリース：従業員1人当たり業績データの追加` → LOC: `新機能リリース\n従業員1人当たり業績データの追加`
  - DS: `【業界最安水準】米国株式信用取引がいよいよスタート！` → LOC: `【業界最安水準】\n米国株式信用取引がいよいよスタート！`

---

### K. Specificity / context 補強

#### K1. ぼかし表現 → 具体補強【頻出】
- ルール: 時期/主体/カテゴリの曖昧さを補い、reader が誰の・いつの話か判別できる形にする。
- 例:
  - DS: `今後相場の行方は？` → LOC: `日本株相場の行方は？`
  - DS: `関税緩和で上昇期待の株は？` → LOC: `関税一時引き下げで上昇期待の株は？`
  - DS: `M7」に匹敵！` → LOC: `M7(マグニフィセント・セブン)」に匹敵！`(略称展開)
  - DS: `3月短観から見た国内景気の現状はどうでしょう？` → LOC: `3月の日銀短観から見た国内景気の現状はどうでしょう？`

#### K2. ロケール誤り(年代・主語・typo)修正【頻出】
- 例:
  - DS: `来年米利下げペース鈍化観測を背景に` → LOC: `26年の米利下げペース鈍化観測を背景に`
  - DS: `ことしに入って…` → LOC: `今年に入って…`(かな→漢字)
  - DS: `5万円500円前後でを推移している` → LOC: `50,500円前後で推移している`(typo + 助詞)
  - DS: `市場の警戒がやや後退したものだと考えられる。` → LOC: `市場の警戒感がやや後退したためと考えられる。`

#### K3. 疑問内容の句尾「。」→「?」転換【中頻度】
- ルール: DS が疑問副詞「か」「どうか」「なるか」等を含む文を「。」で締めている場合、Title/Subject/見出しでは句尾を「?」(または全角「?」per E2)に置換し疑問形を明示。Content 本文では(a) I1 助動詞型で結ぶ、(b) 句尾を「?」に変える、いずれも観察される。
- E2(半角→全角 ?)とは別軸:E2 は記号変換、K3 は「。」→「?」の文末記号置換そのもの。両者を順に適用するケースもあり(K3 → E2)。
- 例:
  - DS: `日経平均は上昇基調が続いており、今週中に60,000円の大台突破なるか。` → LOC: `日経平均は上昇基調が続いており、今週中に60,000円の大台突破なるか?`(2026-04-21 r4 で reproduce)
  - DS: `半導体株はまだ買えるか。` → LOC: `半導体株はまだ買えるか?`(corpus 内同型多数)
- 例外: Content 本文で `~ようだ。` `~見られる。` のような断定/伝聞末尾は「。」維持。疑問副詞「か」を含まない断定文は対象外。

### L. その他

#### L1. 報道発言の鉤括弧化(direct quote 化)【散見】
- ルール: 報道発言は `「」` で括り直接引用化。RL-11 news direct-quote carve-out との整合あり。

#### L2. URL を独立行に分離(button labels)【頻出】
- ルール: button copy + URL は別行に分離してレイアウトを揃える。
- 例: `今すぐ確認：https://...` → `今すぐ確認\nhttps://...`

## 6. Limitations

1. **単一ブランド corpus**: moomoo 証券 JP の 4-person panel 編集スタイルに最適化。他証券・他金融広告主では再検証必要。
2. **DS 3.2 出力に対する pattern**: GPT/Claude/Gemini など他 LLM の翻訳出力には pattern が完全には移植できない可能性(特に B1/B3/F2 のフォーマット系は LLM ごとに既定値が異なる)。
3. **EDM channel に最適化**: Push/SNS/Banner では文字制限が異なるため、A2(語の補完)や J1(改行)は適用条件を要再評価。
4. **観察 n が少ない pattern(n=1, n=2)**: D1, E1, I3, C1 は corpus 内事例が少なく一般化リスクあり。新規 corpus で要検証。
5. **factual 修正(K2)は LLM 改善で消える**: K2 の typo / 年代誤り訂正は LLM 性能向上で頻度が低下する pattern。長期的には A1/A2/F2/G1 等の brand/style 固有 pattern が残ると予想。
6. **Editor 個人差**: 4-person panel の集合的傾向であり、個別 editor のスタイル差は本 expert では分離していない。
